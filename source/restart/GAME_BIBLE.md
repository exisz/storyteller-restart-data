# RESTART Game Bible

## Core Experience

RESTART is a loop mystery game about escaping a system that uses memory, medical authority, broadcast instructions, and paperwork to pre-write people’s deaths.

The player fantasy is not “read the next scene.” It is: survive a repeating route, notice contradictions, carry useful state forward, decide whom to trust, and pay costs to turn knowledge into escape.

## Core Tensions

- Truth vs survival: learning more should often make the current loop more dangerous.
- Trust vs leverage: NPCs may help only when exposed, bargained with, protected, or betrayed.
- Memory vs paperwork: what Linxi remembers can contradict what records say already happened.
- Escape vs rescue: leaving may require abandoning someone, falsifying a death, or becoming complicit.

## Player Verbs

Investigate, hide, bargain, expose, follow, steal, forge, remember, sacrifice, delay, misdirect, confront.

## State Model

Cards are resources/state, not graph nodes. They represent things the player can hold, use, lose, transform, or present in a rite slot.

Flags represent world memory and delayed consequences. A flag is weak unless some later content checks it.

Counters represent pressure or leverage when they meaningfully affect access, cost, or outcome.

## Design Rules

- Every playable node should change situation, knowledge, relationship, pressure, or access.
- Every important choice should leave memory: flag, card, counter, or future lock.
- Optional branches must be real: ignoring them should also matter.
- A node that only says “and then the next thing happened” is suspect.
- Text exists to support play. If prose does not change a decision, pressure, clue, relationship, or state, cut it.
